# GitHub Release Checklist

1. Update `CHANGELOG.md`
2. Bump plugin/app versions if you track them
3. Tag a release in GitHub (e.g., `v0.1.0`)
4. Attach packaged builds (optional)

## Optional: distributing binaries

Consider later:

- Windows: PyInstaller
- macOS: py2app
- Linux: AppImage / flatpak
